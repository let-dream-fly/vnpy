## 1. 写在前面

### 量化交易 vs 手动交易

| 维度 | 手动交易（你现在） | 量化交易（目标） |
|------|------------------|----------------|
| 决策依据 | 主观判断、市场感觉 | 数学模型、历史数据验证 |
| 执行效率 | 人工操作，有延迟 | 程序自动下单，毫秒级 |
| 情绪影响 | 受情绪干扰（贪婪/恐惧） | 严格执行规则，不受情绪影响 |
| 回测验证 | 难以精确评估历史表现 | 可在历史数据上精确量化绩效 |
| 适合场景 | 基本面分析、长期价值投资 | 趋势跟踪、统计套利、多因子选股 |

### 关于基金的说明

> **重要提示**：VeighNa 主要针对**可日内交易的证券品种**（A股、ETF、期货等）设计。
>
> - **股票型/混合型基金（非 ETF）**：T+1 申购赎回，无实时行情撮合，**不适合量化策略**
> - **ETF（交易型开放式基金）**：在二级市场像股票一样交易，**完全支持量化策略** ✅
> - **A 股股票**：完全支持 ✅
>
> 本指引重点围绕 **A股股票 + ETF** 展开。

### 关键名词速查

| 名词 | 含义 |
|------|------|
| **vt_symbol** | 平台统一合约标识符，格式为 `代码.交易所`，如 `000001.SSE`（上交所平安银行） |
| **Bar** | K 线数据，包含某时间段内的开高低收量 |
| **Tick** | 逐笔行情数据，每次价格变动推送一次 |
| **Gateway** | 交易接口，连接到券商/交易所的桥梁 |
| **datafeed** | 数据服务，提供历史行情数据 |
| **回测** | 用历史数据模拟策略在过去的表现 |
| **仿真** | 用真实行情但虚拟资金模拟交易 |

```
┌─────────────────────────────────────────────────────────────────┐
│                        你写的策略代码                            │

├─────────────────────────────────────────────────────────────────┤

│     因子工程(dataset)  ML模型(model)  策略(strategy)  投研(lab)  │
├─────────────────────────────────────────────────────────────────┤
│                      核心框架层 (vnpy.trader)                    │
│   EventEngine（事件总线）  MainEngine（主引擎）  OmsEngine（OMS）│
├─────────────────────┬───────────────────────────────────────────┤
│  基础设施层          │  工具与算法层                              │
│  数据库(SQLite等)   │  BarGenerator   ArrayManager(指标库)      │
│  数据源(TuShare等)  │  OptimizationSetting  遗传算法优化器       │
│  配置(vt_setting)   │                                           │
├─────────────────────┴───────────────────────────────────────────┤
│                    交易接口层 (Gateway)                          │
│  期货: CTP    股票: XTP / 华鑫奇点    仿真: PaperAccount        │
│  数据行情: RQData / 迅投研 / TuShare / Wind                     │
└─────────────────────────────────────────────────────────────────┘
```

### 数据流向

```
交易所行情 → Gateway → EventEngine → 策略 on_bar()/on_tick()
                                  ↓
                            策略信号生成
                                  ↓
                         buy/sell/short/cover
                                  ↓
                   MainEngine → Gateway → 券商 → 成交
                                  ↓
                      成交回报 → on_trade() → 更新持仓
```

**Step 3：代码方式下载（批量下载多只股票）**

```python
# download_stocks.py — 批量下载股票日线数据
from vnpy.trader.setting import SETTINGS
from vnpy.trader.database import get_database
from vnpy.trader.datafeed import get_datafeed
from vnpy.trader.object import HistoryRequest
from vnpy.trader.constant import Exchange, Interval
from datetime import datetime

# 配置数据源
SETTINGS["datafeed.name"] = "tushare"
SETTINGS["datafeed.password"] = "你的TuShare_Token"

# 要下载的股票列表（合约代码.交易所）
STOCKS = [
    ("000001", Exchange.SZSE),   # 平安银行
    ("000300", Exchange.SSE),    # 沪深300指数
    ("510300", Exchange.SSE),    # 沪深300ETF
    ("600036", Exchange.SSE),    # 招商银行
    ("600519", Exchange.SSE),    # 贵州茅台
]

datafeed = get_datafeed()
database = get_database()

for symbol, exchange in STOCKS:
    req = HistoryRequest(
        symbol=symbol,
        exchange=exchange,
        interval=Interval.DAILY,       # 日线
        start=datetime(2020, 1, 1),
        end=datetime(2024, 12, 31),
    )
    bars = datafeed.query_bar_history(req)
    if bars:
        database.save_bar_data(bars)
        print(f"已下载 {symbol}.{exchange.value}: {len(bars)} 根日线")
    else:
        print(f"下载失败: {symbol}.{exchange.value}")
```

## 6. 第一个策略：双均线策略（从代码到回测）

### 6.1 策略开发：继承 CtaTemplate

所有 CTA 策略必须继承 `CtaTemplate`，这个基类定义了策略的生命周期和交易接口。

**创建策略文件** `my_strategies/double_ma_stock.py`：

```python
"""
双均线股票策略 — 新手入门示例
适用品种：A股 / ETF（日线级别）
逻辑：快线均值 > 慢线均值 → 持仓；快线 < 慢线 → 清仓
"""
from vnpy_ctastrategy import CtaTemplate, StopOrder
from vnpy.trader.object import TickData, BarData, TradeData, OrderData
from vnpy.trader.utility import BarGenerator, ArrayManager


class DoubleMaStockStrategy(CtaTemplate):
    """双均线股票/ETF策略"""
    author = "小白入门"

    # ---- 策略参数（UI中可调整，支持参数优化）----
    fast_window = 5      # 快线周期（日）
    slow_window = 20     # 慢线周期（日）
    parameters = ["fast_window", "slow_window"]

    # ---- 策略变量（运行状态，自动持久化）----
    fast_ma = 0.0        # 快线当前值
    slow_ma = 0.0        # 慢线当前值
    variables = ["fast_ma", "slow_ma"]

    def on_init(self) -> None:
        """初始化：加载历史数据预热指标"""
        # ArrayManager 存储最近 N 根 Bar 的数据，用于计算指标
        self.am = ArrayManager(size=self.slow_window + 10)
        # 加载 30 天历史数据驱动 on_bar 预热 ArrayManager
        self.load_bar(30)
        self.write_log("策略初始化")

    def on_start(self) -> None:
        self.write_log("策略启动，开始监听行情")

    def on_stop(self) -> None:
        self.write_log("策略停止")

    def on_bar(self, bar: BarData) -> None:
        """每根 K 线结束时触发（日线策略每天收盘后触发一次）"""
        self.cancel_all()      # 撤销上一轮未成交的委托

        am = self.am
        am.update_bar(bar)     # 把新的 Bar 数据推入数组

        if not am.inited:      # 数据不足，指标不可靠，跳过
            return

        # 计算移动平均线
        self.fast_ma = am.sma(self.fast_window, array=False)
        self.slow_ma = am.sma(self.slow_window, array=False)

        # 交叉信号判断
        cross_over = self.fast_ma > self.slow_ma    # 金叉：快线在慢线之上
        cross_below = self.fast_ma < self.slow_ma   # 死叉：快线在慢线之下

        # 开仓逻辑
        if cross_over and self.pos == 0:
            # 金叉且无仓位 → 买入
            # A股最小单位：1手=100股，price 用当前收盘价委托
            self.buy(bar.close_price, 1)

        elif cross_below and self.pos > 0:
            # 死叉且持有多仓 → 卖出清仓
            self.sell(bar.close_price, abs(self.pos))

        self.put_event()       # 刷新 UI 显示策略状态

    def on_order(self, order: OrderData) -> None:
        """委托状态变化回调"""
        pass

    def on_trade(self, trade: TradeData) -> None:
        """成交回报回调"""
        self.put_event()

    def on_stop_order(self, stop_order: StopOrder) -> None:
        """停止单回调"""
        pass
```

### 6.2 策略关键接口说明

```
策略生命周期（按调用顺序）：

on_init()    ← 初始化，加载历史数据，计算初始指标
    │
    ▼
on_start()   ← 策略启动，开始接收实时行情
    │
    ▼ 每根 Bar 结束（日线: 每天15:00后触发）
on_bar(bar)  ← 核心逻辑：计算信号 → 下单
    │
    ▼
on_order()   ← 委托状态变化（提交、成交、撤销）
on_trade()   ← 成交确认
    │
    ▼
on_stop()    ← 策略停止（手动停止或程序退出）
```

**四个核心下单指令**：

| 方法 | 动作 | 股票场景 |
|------|------|---------|
| `self.buy(price, volume)` | 买入开多 | 买入股票 |
| `self.sell(price, volume)` | 卖出平多 | 卖出持有的股票 |
| `self.short(price, volume)` | 卖空开空 | 融券做空（股票需融券权限） |
| `self.cover(price, volume)` | 买入平空 | 归还融券 |

> 普通股票账户（无融券）只用 `buy` 和 `sell`。

### 6.3 代码方式运行回测

```python
# backtest_stock.py — 在命令行运行回测，无需 GUI
from vnpy_ctabacktester.engine import BacktestingEngine
from vnpy.trader.constant import Interval
from datetime import datetime

# 导入你的策略
from my_strategies.double_ma_stock import DoubleMaStockStrategy

# 创建回测引擎
engine = BacktestingEngine()
engine.set_parameters(
    vt_symbol="510300.SSE",        # 沪深300ETF
    interval=Interval.DAILY,       # 日线（股票常用）
    start=datetime(2020, 1, 1),    # 回测开始
    end=datetime(2024, 12, 31),    # 回测结束
    rate=0.0003,                   # 佣金费率（ETF约万三）
    slippage=0.01,                 # 滑点（1分价差）
    size=100,                      # 每手股数（A股ETF为100份）
    pricetick=0.001,               # 最小价格变动
    capital=100_000,               # 初始资金 10 万元
)

# 添加策略，传入参数字典
engine.add_strategy(DoubleMaStockStrategy, {
    "fast_window": 5,
    "slow_window": 20,
})

# 运行回测
engine.load_data()
engine.run_backtesting()
df = engine.calculate_result()
stats = engine.calculate_statistics(df)
engine.show_chart(df)              # 显示交互式资金曲线图

# 打印关键指标
```

### 7.6 无 GUI 服务器部署（进阶）

适合在 Linux 服务器上 7x24 小时自动运行（不包含非交易时段）：

```python
# run_server_stock.py — 股票策略守护进程
import multiprocessing
import sys
from time import sleep
from datetime import datetime, time

from vnpy.event import EventEngine
from vnpy.trader.setting import SETTINGS
from vnpy.trader.engine import MainEngine

from vnpy_xtp import XtpGateway
from vnpy_ctastrategy import CtaStrategyApp

SETTINGS["log.active"] = True
SETTINGS["log.level"] = 20
SETTINGS["log.console"] = True
SETTINGS["datafeed.name"] = "tushare"
SETTINGS["datafeed.password"] = "你的Token"

# XTP 账户配置
XTP_SETTING = {
    "用户ID": "你的证券账号",
    "密码": "你的密码",
    "客户号": 1,
    "行情服务器地址": "行情服务器IP",
    "行情服务器端口": 6001,
    "交易服务器地址": "交易服务器IP",
    "交易服务器端口": 6001,
    "产品信息": "vnpy",
    "授权码": "授权码",
}

# A股交易时段
MORNING_START = time(9, 25)    # 集合竞价开始
MORNING_END   = time(11, 35)
AFTERNOON_START = time(12, 55)
AFTERNOON_END   = time(15, 10)


def check_trading_period() -> bool:
    now = datetime.now()
    # 仅工作日交易（简化判断，实际需加节假日判断）
    if now.weekday() >= 5:
        return False
    t = now.time()
    return (
        (MORNING_START <= t <= MORNING_END) or
        (AFTERNOON_START <= t <= AFTERNOON_END)
    )


def run_child():
    ee = EventEngine()
    me = MainEngine(ee)
    me.add_gateway(XtpGateway)
    cta_engine = me.add_app(CtaStrategyApp)

    me.connect(XTP_SETTING, "XTP")
    sleep(10)

    cta_engine.init_engine()
    cta_engine.init_all_strategies()
    sleep(30)
    cta_engine.start_all_strategies()

    while True:
        sleep(10)
        if not check_trading_period():
            me.close()
            sys.exit(0)


def run_parent():
    child = None
    while True:
        trading = check_trading_period()
        if trading and child is None:
            child = multiprocessing.Process(target=run_child)
            child.start()
        if not trading and child is not None:
            if not child.is_alive():
                child = None
        sleep(5)


if __name__ == "__main__":
    run_parent()
```

## 9. 常见问题与避坑指南

### Q1：我的股票账户能直接连 VeighNa 吗？

不一定。VeighNa 通过「交易接口（Gateway）」连接券商，需要以下条件：

1. 你的券商支持开放 API（如 XTP、TORA、EMT）
2. 你向券商申请开通 API 权限（部分需要资产门槛，如 50 万）
3. 券商提供服务器地址和授权码

**建议流程**：先用仿真账户测试策略 → 验证稳定后再申请实盘 API

### Q2：TuShare 免费吗？数据质量怎么样？

TuShare 采用积分制：注册即获得一定积分，可访问基础股票日线数据；更高频率/更多品种需要充值积分（付费）。

对于新手入门回测，免费积分已足够。

| 数据类型 | 所需积分 | 说明 |
|---------|---------|------|
| 股票日线行情 | 120 | 新手注册即可获得 |
| 股票分钟线 | 2000+ | 需付费 |
| ETF日线 | 120 | 注册即可 |
| 基金净值 | 600+ | 需付费或积分 |

### Q3：股票和 ETF 回测时需要注意什么？

| 注意点 | 说明 |
|--------|------|
| A股涨跌停板 | 回测引擎不会自动处理，需在策略中限制委托价格 |
| T+1 限制 | 当日买入不能当日卖出（ETF 也是 T+1），回测引擎不自动处理，需注意 |
| 手续费 | A股佣金约万三，ETF 约万一，印花税仅卖出收取 0.1% |
| 滑点 | 日线策略建议设置 0.01~0.02 元 |
| 合约乘数 size | A股/ETF 设为 `100`（每手100股/份） |

### Q4：我关注的基金如何配置？

如果你关注的是 **ETF 基金**（如沪深300ETF 510300），直接按股票方式操作即可。

如果你关注的是 **场外基金**（如各大基金公司的普通开放式基金），这类基金：
- 不在交易所实时交易，无 Tick/Bar 数据
- 一般 T+1 申购、T+2/T+3 赎回
- 不适合量化策略，只能做定期调仓的定投模型

### Q5：如何知道策略数据是否正确加载？

在策略 `on_init` 中加日志：

```python
def on_init(self) -> None:
    self.am = ArrayManager(size=self.slow_window + 10)
    self.load_bar(30)
    self.write_log(f"初始化完成，数据点数: {len(self.am.close_array)}")
```

或者在回测结束后检查：

```python
engine.load_data()
print(f"加载数据条数: {len(engine.history_data)}")
```

### Q6：为什么回测图表是空的？

最常见原因：数据库里没有该合约的数据。解决方案：
1. 先在【应用 → 数据管理】中确认数据已下载
2. 检查合约代码格式（`510300.SSE` 而非 `510300`）
3. 检查回测的起止日期是否在数据范围内
